# Roadmap Backend Developer Junior (2026)

Um plano estruturado e atualizado focado nas demandas reais do mercado para vagas de **Backend Júnior**, com ênfase em stacks modernas, boas práticas, cloud, APIs e observabilidade.

---

## Visão Geral do Roadmap

1. Fundamentos da Programação  
2. Lógica + Estruturas de Dados  
3. Git e GitHub  
4. Backend com linguagem principal  
5. APIs REST  
6. Banco de Dados (SQL + NoSQL)  
7. Docker  
8. Cloud Básica  
9. Testes  
10. Observabilidade e Logs  
11. CI/CD  
12. Projeto Real + Portfólio  
13. Deploy, Currículo e Entrevistas  

---

## Stack Mais Estratégica para o Mercado Atual

- **Linguagem:** Java  
- **Framework:** Spring Boot  
- **Banco de Dados:** PostgreSQL  
- **Containers:** Docker  
- **Cloud:** Amazon Web Services (AWS)  
- **Versionamento:** Git + GitHub  
- **Observabilidade:** Grafana + Prometheus (ou Datadog)  

Essa stack é uma das mais solicitadas em bancos, fintechs, consultorias e grandes empresas no Brasil.

## Fases do Roadmap

### FASE 1 — Fundamentos da Programação (2–4 semanas)
**Objetivo:** Desenvolver o raciocínio lógico.  
**Tópicos:** Variáveis, Condições, Loops, Funções, Arrays/Listas, Objetos, Tratamento de Erros.  

**Exercícios:** Calculadora, Jogo da Forca, Sistema Bancário Simples, Conversor de Moedas.  
**Plataformas:** [Exercism](https://exercism.org), [Beecrowd](https://www.beecrowd.com.br), [LeetCode](https://leetcode.com)

### FASE 2 — Git e GitHub (1 semana)
**Tópicos:** `git init`, `clone`, `add`, `commit`, `push`, Branch, Merge, Pull Request.  
**Entrega:** Subir todos os exercícios da Fase 1 no GitHub.

### FASE 3 — Java + Backend (6–8 semanas)
- **Java:** Classes, Objetos, Herança, Interfaces, Collections, Streams, Exceptions, Maven/Gradle.  
- **Spring Boot:** Controllers, Services, Repositories, Injeção de Dependência, DTOs, Validação.

### FASE 4 — APIs REST (3 semanas)
**Tópicos:** HTTP, REST, JSON, CRUD, Status Codes, Headers, Query Params.  
**Ferramentas:** Postman ou Insomnia.  
**Projeto:** API de Usuários, Produtos e Pedidos.

### FASE 5 — Banco de Dados (4 semanas)
- **SQL (Obrigatório):** PostgreSQL — SELECT, INSERT, UPDATE, DELETE, JOINs, GROUP BY, Índices.  
- **NoSQL:** MongoDB (Documentos, Collections e Modelagem).

### FASE 6 — Docker (2 semanas)
**Tópicos:** Containers, Dockerfile, Docker Compose, Volumes, Networks.  
**Projeto:** Rodar API + PostgreSQL + Redis via Docker Compose.

### FASE 7 — Cloud Básica (3 semanas)
**Foco:** Amazon Web Services (AWS)  
**Serviços:** EC2, S3, RDS, IAM, CloudWatch.

### FASE 8 — Testes (2 semanas)
- Testes unitários e de integração  
**Ferramentas:** JUnit + Mockito

### FASE 9 — Observabilidade (2 semanas)
**Tópicos:** Logs, Métricas, Traces, Health Checks.  
**Ferramentas:** Grafana, Prometheus, Datadog.

### FASE 10 — CI/CD (2 semanas)
**Ferramenta:** GitHub Actions (Build, Test, Deploy).

### FASE 11 — Segurança Básica (1 semana)
**Tópicos:** JWT, Autenticação, Hash de senha, OAuth básico, CORS.

### FASE 12 — Projeto Final (4 semanas)
Desenvolva um sistema real (E-commerce, Delivery, Gestão Financeira, etc.) contendo:
- API REST + PostgreSQL
- Docker + Docker Compose
- JWT
- Logs e Métricas
- Deploy na Cloud
- CI/CD
- Swagger/OpenAPI
- README profissional

### FASE 13 — Portfólio e Mercado
- **GitHub:** Repositórios organizados, READMEs bem escritos e commits frequentes.  
- **LinkedIn:** Poste projetos, aprendizados e deploys.
